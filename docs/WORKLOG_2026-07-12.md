# Work log — 12 July 2026

This document records the product and operational context established during the first complete Paper Picture MVP build. It intentionally excludes secrets, private database rows, browser sessions, raw feedback, and registrar credentials.

## Product direction

The original idea was refined into a progressive visual research game for computer-graphics and digital-geometry papers. A player sees one figure, chooses among institution or country answers, and may reveal up to two more figures for fewer possible points. The long-term concept includes guessing research groups, universities, countries, and authors across venues and journals.

The first release deliberately changed from “last year’s SIGGRAPH papers” to a license-first collection because paper availability does not imply permission to redistribute figures. No fictional substitute content was accepted.

## Rights-first collection

`Open Graphics Collection 01 · v1.0` was frozen with six real papers and eighteen real figures. Every included article explicitly states CC BY 4.0 coverage for the article and included images unless separately credited. Selected captions were checked for separate third-party credits, affiliations were verified, and publisher-provided images were kept unmodified except for responsive browser sizing.

Approved papers:

1. Goal-adaptive Meshing of Isogeometric Kirchhoff–Love Shells
2. Functional maps regularization for high quality mesh morphing applied to shape registration in computed tomography
3. Topology-inclusive aerodynamic shape optimisation using a cellular automata parameterisation
4. Be water my friend: mesh assimilation
5. Higher-order block-structured hex meshing of tubular structures
6. CADGCL: unsupervised retrieval of CAD models via boundary representations

The rights record, source URLs, figure numbers, license evidence, modifications, and asset hashes are stored in `data/RIGHTS_AUDIT.md` and `data/rights-evidence.json`. Tests fail if any shipped image no longer matches its recorded hash.

## Game implementation

- Six randomized rounds per complete game
- One figure shown initially; up to three figures per paper
- Correct scoring of 100 / 70 / 40 based on figures seen
- Zero points for an incorrect answer
- Server recalculation of attempts and final score
- Assisted-round tracking when a source link is opened before answering
- Clear paper, author, figure, source, license, and modification attribution
- Responsive layout, keyboard-oriented controls, reduced-motion CSS, and descriptive image alt text

The score remains casual rather than competition-grade because the client receives the correct option. This limitation is documented in the product and roadmap.

## Identity, profiles, and database

ChatGPT sign-in identity headers are consumed only by server code. The normalized email is transformed into a pseudonymous HMAC key using a hosted `PROFILE_ID_SECRET`; the raw email and ChatGPT tokens are not written to gameplay tables.

D1 migrations created:

- `profiles`
- `game_sessions`
- `round_attempts`
- `feedback`
- `rate_limits`

Implemented player capabilities:

- Private profile
- Editable display name
- Games played, best score, accuracy, reveals, assisted answers, and recent history
- Full profile and related-data deletion
- Saved randomized paper order and immutable collection version per session
- Idempotent completion and duplicate-attempt protection
- Private no-store API responses and per-action rate limits

Retention implemented:

- Incomplete sessions: 7 days
- Feedback: 365 days
- Rate-limit counters: approximately 2 days
- Completed profile/history: until player deletion

## Feedback and owner operations

An authenticated feedback form accepts gameplay, difficulty, metadata, copyright, accessibility, bug, privacy, and other reports. The owner-only `/admin/feedback` route supports filtering, status changes (`new`, `reviewing`, `resolved`), and CSV export. Non-owner users receive a not-found response rather than confirmation that the admin surface exists.

The application never places feedback contents or raw emails in operational error logs. Database exports and feedback CSVs are private operational data and are excluded from Git.

## Privacy and security hardening

- Privacy notice describing identity, storage, retention, deletion, and rights contact
- Test guide describing scoring, assisted play, conduct, and success criteria
- HMAC-derived player keys
- Server-side ownership checks on every private record operation
- Prepared D1 statements and database uniqueness constraints
- Request rate limits
- `Cache-Control: private, no-store` on private APIs
- HSTS, CSP, frame denial, nosniff, permissions policy, referrer policy, and cross-origin opener policy
- Health endpoint that reports only immutable collection counts
- Admin routes protected by the hosted owner email configuration

## Brand and public web metadata

The product name was finalized as **Paper Picture** and the domain as **paperpicture.net**. A social preview image, favicon, page titles, descriptions, canonical URLs, route-specific indexing rules, robots file, and sitemap were added.

Private profile, feedback, admin, and API surfaces are excluded from indexing. The sitemap contains only the home page, privacy notice, and tester guide.

## Domain and deployment

The domain was registered at METUnic and attached to the existing OpenAI Sites project. The registrar zone contains two apex A records supplied by Sites plus the required OpenAI/Cloudflare TXT verification records. METUnic nameservers are selected. Public DNS, route activation, and the managed TLS certificate were verified.

Final observed hosting state:

- Custom domain: active
- Provider route: active
- SSL: active
- Primary URL: <https://paperpicture.net>
- Deployed site version: 9

The owner-only access policy was intentionally preserved. An external friend cannot test until the owner explicitly changes the audience policy.

## Decisions that should not be lost

- Real content only; no fictional fallback.
- Image rights are a hard launch gate, not a cleanup task.
- The collection is immutable once released.
- Profiles are private and pseudonymous in application storage.
- Scores are casual; no leaderboard or prizes.
- The site stays owner-only until manual acceptance and an explicit audience decision.
- Production secrets and player data never belong in GitHub.
- Expand by creating new reviewed collections, not by weakening license checks.
